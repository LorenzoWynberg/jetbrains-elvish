Knowledge Architecture

Three layers of documentation

Learnings

docs/learnings/
  • • Topic-based files
  • • Gotchas in each file
  • • Read first every time

Activity Logs

docs/activity/
  • • Daily session notes
  • • Decisions made
  • • Last 2 in prompt

Progress

scripts/ralph/progress.txt
  • • Story history
  • • Per-story learnings
  • • Architecture notes

Learning from Everything

On Success

  • → Add patterns to learnings
  • → Update activity log
  • → Set passes: true
  • → Create PR and merge

On Blocked

  • → Add gotchas to learnings
  • → Document what failed
  • → DON'T update prd.json
  • → Retry or skip
